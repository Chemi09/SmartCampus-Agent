# EduSovereign AI - Frontend React\n\n## 📋 Vue d'ensemble\n\n**EduSovereign AI** est une plateforme ERP scolaire complète avec CRM intégré et support pour un futur agent IA. L'application est conçue pour les écoles en Afrique (Kinshasa) et offre une gestion complète des élèves, notes, paiements et communication parents-école.\n\n## 🎯 Fonctionnalités principales\n\n### 1. **Authentification & Rôles**\n- 4 rôles utilisateur : Admin, Professeur, Parent, Élève\n- Système de connexion sécurisé\n- Dashboards personnalisés par rôle\n\n### 2. **ERP Scolaire**\n- **Gestion des élèves** : CRUD complet (ajouter, modifier, supprimer, lister)\n- **Gestion des notes** : Ajout de notes avec calcul automatique des moyennes\n- **Gestion des classes** : Organisation par classes\n\n### 3. **CRM École-Parents**\n- **Suivi des paiements** : Statuts (payé, en attente, en retard, partiel)\n- **Système de relance** : Notifications automatiques pour paiements en retard\n- **Messagerie** : Communication bidirectionnelle parents-école type WhatsApp\n\n### 4. **Dashboard**\n- Statistiques en temps réel\n- Graphiques (Recharts) : moyennes, paiements, statuts\n- Alertes et notifications\n- Activité récente\n\n### 5. **Paramètres**\n- Gestion du profil utilisateur\n- Sécurité (changement de mot de passe)\n- Préférences de notifications\n- Confidentialité des données\n\n## 🏗️ Architecture & Structure\n\n```\nclient/src/\n├── components/\n│   ├── common/           # Composants réutilisables\n│   │   ├── Sidebar.tsx\n│   │   ├── Navbar.tsx\n│   │   ├── DashboardLayout.tsx\n│   │   ├── StatCard.tsx\n│   │   └── DataTable.tsx\n│   └── ui/              # Composants shadcn/ui\n├── contexts/\n│   └── AuthContext.tsx  # Gestion de l'authentification et rôles\n├── hooks/\n│   └── useApi.ts        # Hooks API personnalisés\n├── pages/\n│   ├── Login.tsx\n│   ├── Dashboard.tsx\n│   ├── Settings.tsx\n│   ├── ERP/\n│   │   ├── Students.tsx\n│   │   └── Grades.tsx\n│   └── CRM/\n│       ├── Payments.tsx\n│       └── Feedback.tsx\n├── types/\n│   └── index.ts         # Types TypeScript\n├── utils/\n│   └── format.ts        # Utilitaires (devises, dates, etc.)\n└── App.tsx              # Routes et configuration\n```\n\n## 🚀 Démarrage rapide\n\n### Prérequis\n- Node.js 18+\n- pnpm (ou npm/yarn)\n\n### Installation\n\n```bash\n# Installer les dépendances\npnpm install\n\n# Lancer le serveur de développement\npnpm dev\n```\n\nL'application sera disponible à `http://localhost:3000`\n\n### Comptes de démonstration\n\n```\nEmail: admin@edusovereign.cd\nMotdepasse: demo\nRôle: Administrateur\n\nEmail: teacher@edusovereign.cd\nMotdepasse: demo\nRôle: Professeur\n\nEmail: parent@edusovereign.cd\nMotdepasse: demo\nRôle: Parent\n\nEmail: student@edusovereign.cd\nMotdepasse: demo\nRôle: Élève\n```\n\n## 🔌 Intégration Backend\n\n### Points d'intégration API\n\nLe frontend est prêt à se connecter à un backend. Voici les points clés :\n\n#### 1. **Authentification** (`client/src/contexts/AuthContext.tsx`)\n```typescript\n// Remplacer le mock par :\nconst response = await axios.post('/api/auth/login', { email, password });\n```\n\n#### 2. **Hooks API** (`client/src/hooks/useApi.ts`)\n```typescript\n// Remplacer les mock data par :\nconst { data } = useApi('/api/students');\nconst { data: grades } = useApi('/api/grades');\nconst { data: payments } = useApi('/api/payments');\n```\n\n#### 3. **Configuration Axios**\n```typescript\n// Créer un fichier client/src/lib/axios.ts\nimport axios from 'axios';\n\nconst instance = axios.create({\n  baseURL: process.env.VITE_API_URL || 'http://localhost:3001/api',\n  timeout: 10000,\n  headers: {\n    'Content-Type': 'application/json'\n  }\n});\n\n// Ajouter l'intercepteur pour le JWT\ninstance.interceptors.request.use((config) => {\n  const token = localStorage.getItem('token');\n  if (token) {\n    config.headers.Authorization = `Bearer ${token}`;\n  }\n  return config;\n});\n\nexport default instance;\n```\n\n### Endpoints attendus\n\n```\nPOST   /api/auth/login\nGET    /api/students\nPOST   /api/students\nPUT    /api/students/:id\nDELETE /api/students/:id\n\nGET    /api/grades\nPOST   /api/grades\nPUT    /api/grades/:id\nDELETE /api/grades/:id\n\nGET    /api/payments\nPOST   /api/payments\nPUT    /api/payments/:id\nDELETE /api/payments/:id\n\nGET    /api/messages\nPOST   /api/messages\nGET    /api/conversations\n\nGET    /api/dashboard/stats\n```\n\n## 🛠️ Stack technique\n\n- **React 19** - Framework UI\n- **Vite** - Build tool\n- **TypeScript** - Type safety\n- **Tailwind CSS 4** - Styling\n- **shadcn/ui** - Composants UI\n- **React Router (Wouter)** - Routing\n- **React Hook Form + Zod** - Formulaires et validation\n- **Axios** - HTTP client\n- **Recharts** - Graphiques\n- **Lucide React** - Icônes\n- **Sonner** - Notifications toast\n\n## 📱 Responsive Design\n\nL'application
